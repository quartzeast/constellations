一个 React 代码库中的组件可以自然的落入三类：无状态或展示性组件（stateless or presentational components），有状态组件（stateful components）或结构化组件（structural components）。这里的自然的含义是在编写代码时，我们不应该强制性的将组件变成这三类中的一种，顺其自然就好。

这些组件仍然是我们编写的常规 React 组件，但是当我们可以从概念层面上将其分为这三类。虽然还有其它分类方式，但是这三类分类方式在我看来是比较合理的。

- 无状态或展示性组件：这些组件没有状态，它们通常会接收一些 props，然后简单的展示这些数据或其它内容，因此是展示性的。这些通常是小组件并且可重用
- 有状态组件：有状态组件就是拥有状态的组件。组件有状态仍然可以被重用
- 结构性组件：结构性组件可以视为应用的页面、布局、屏幕，这些通常是将许多较小的组件组合在一起的结果。结构性组件可能是比较大的且不可重用的组件，但也有可能可以被重用。它们负责为应用提供某种解构，如页面或布局
