组件的生命周期其实指的是组件实例的生命周期，因为只有真正的组件实例才能经历生命周期。但是为了方便一般简称组件的生命周期。

组件的生命周期指某个组件实例随时间可能经历的不同阶段。

## 组件的挂载/初次渲染

组件生命周期的第一阶段是组件实例的挂载/初始渲染。在这个阶段组件被初次渲染，并创建新状态和 Props，这个阶段可以比喻成组件的诞生。

## 组件的重新渲染

一旦组件被渲染在 UI 上，就可无限次的重新渲染。状态更新时 React 会重新渲染。组件接收的 props 改变、父组件重新渲染或 context 改变时也会发生重新渲染。重新渲染阶段是可选的，因此不是所有组件都会重新渲染。有些组件只挂载然后立即卸载。

## 组件的卸载

当一个组件不再被需要时，组件就会被卸载。组件（连同 state 和 Props）被销毁并移除。可能发生在用户导航到 UI 的另一个部分或新页面时，或完全关闭应用时。稍后可以挂载同一组件的新实例，但此特定实例已消失，该组件已被卸载。理解组件生命周期很重要，因为我们可以在不同阶段使用 React Hooks Hook 进这个生命周期。因此，我们可以定义代码在这些时间点执行。
